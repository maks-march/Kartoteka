1. Введение

Картотека промышленных объектов — это информационная система для учета и анализа промышленных объектов, их автоматизированных систем управления, а также участников рынка (вендоров, интеграторов, поставщиков).

2. Быстрый старт

2.1. Вход в систему

  1. Откройте браузер и перейдите по адресу вашей системы
  2. Введите имя пользователя и пароль
  3. Нажмите кнопку "Войти"

Элементы интерфейса

  • Верхнее меню — основные разделы системы
  • Боковая панель — фильтры и настройки
  • Имя пользователя — меню профиля и выход

3. Архитектура системы

Система построена на современном технологическом стеке с использованием лучших практик веб-разработки. Основой является фреймворк Django с базой данных PostgreSQL.

3.1. Технологический стек

Backend

Python 3.9+ Django 4.x PostgreSQL 16+ Gunicorn

Основной язык — Python, веб-фреймворк Django, БД PostgreSQL.

Frontend

Bootstrap 5 HTML5/Django Templates JavaScript

Адаптивный интерфейс на Bootstrap, интерактивные элементы.

Инфраструктура

Git Pip Virtualenv

Контроль версий, управление зависимостями, изоляция окружения.

3.3. Модель данных

Основные сущности

Сущность Описание Ключевые поля
Object Промышленные объекты name, hierarchy_level, parent, legal_entity
Participant Участники рынка name, inn, participant_type, is_partner
AutomatedSystem Автоматизированные системы name, system_class, object, vendor

Иерархическая структура

Объекты могут иметь родительские и дочерние связи, образуя иерархию:

  • LEVEL_1 — Предприятия/НПЗ (корневые объекты)
  • LEVEL_2 — Цеха/Производства (дочерние)
  • LEVEL_3 — Установки/Агрегаты (дочерние)

3.4. Безопасность

Ролевая модель

  • Администратор — полный доступ, включая удаление
  • Редактор — создание и редактирование всех записей
  • Наблюдатель — только просмотр

Механизмы защиты

  • Аутентификация — вход по логину/паролю
  • Декораторы — @login_required, @editor_required
  • CSRF защита — встроенный middleware Django
  • Аудит действий — логирование всех изменений

3.5. Производительность

Оптимизация запросов

  • select_related — для ForeignKey (JOIN)
  • prefetch_related — для обратных связей
  • Индексы — в БД для часто используемых полей

Кэширование

Наиболее ресурсоемкие операции кэшируются:

  • Граф связей — кэшируется на 1 час
  • API данные — кэшируются для быстрого доступа
Админ-панель: Доступ к панели администратора Django осуществляется по адресу /admin/. Там можно управлять пользователями, группами и всеми данными системы.

4. Django Admin (Панель управления)

Django Admin — это встроенная панель администратора для управления всеми данными системы. Доступ имеют только пользователи с правами staff.

4.1. Вход в админку

  1. Откройте браузер и перейдите по адресу http://ваш-сайт/admin/
  2. Введите имя пользователя (например, admin) и пароль 12345678
  3. Нажмите кнопку "Log in"

После входа открывается главная страница со списком всех моделей:

10. Часто задаваемые вопросы

Как зайти в админку? ▼
Перейдите по адресу /admin/ (например, http://127.0.0.1:8000/admin/). Используйте логин admin и пароль 12345678.
Как посмотреть историю изменений? ▼
В админке откройте раздел Журнал аудита (Audit logs). Там видно все изменения с датой и пользователем.
Как создать нового пользователя? ▼
В админке перейдите Authentication and Authorization → Users, нажмите + Add, заполните username и пароль.
